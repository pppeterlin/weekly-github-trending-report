## 三、Project Detailed Introduction

### 1. [zilliztech/claude-context](https://github.com/zilliztech/claude-context)（排名：1）⭐ 領域精選

- **核心功能**：為 Claude Code 設計的代碼搜索 MCP 工具，將整個代碼庫建立向量索引，使 Claude Code 的每次查詢都能以語義搜索方式命中整個 codebase，徹底突破 token 窗口的限制。
- **推薦原因**：直接強化 Agent 的長期記憶與代碼上下文管理能力——對於數據工程與生物信息管線開發場景，超大型代碼庫的語義定位一直是主要痛點；此工具讓 Claude Code 能精準理解項目全貌而無需反覆粘貼代碼片段。
- **技術亮點**：基於 Zilliz/Milvus 向量數據庫；MCP 協議原生整合；語義代碼搜索精度優於關鍵詞搜索；已在 GitHub 累積 9.3k stars 並持續上升。

---

### 2. [HKUDS/RAG-Anything](https://github.com/HKUDS/RAG-Anything)（排名：9）⭐ 領域精選

- **核心功能**：全模態 All-in-One RAG 框架，統一支援文字文件、表格數據、圖像、音訊、影片等多種模態的知識檢索，構建跨模態的統一知識問答系統。
- **推薦原因**：生物信息學研究場景中，知識往往分散於 PDF 論文、基因數據表格、顯微鏡圖像與實驗音訊記錄之中；RAG-Anything 的全模態整合能力可大幅降低構建多源生物知識庫的工程複雜度。
- **技術亮點**：#RAG #Document-processing 雙核心標籤；支援多模態向量化與統一索引；18.6k stars 本週快速攀升，2.1k forks 顯示高度採用；香港大學數據智能實驗室出品，學術背景扎實。

- **核心功能**：OpenAI 官方輕量級多 Agent 工作流框架，支援 Agent 委派、工具整合（含 MCP 原生支援）、安全護欄、人工介入點、對話歷史管理與語音 Agent 等生產級特性。
- **推薦原因**：連續多週上榜，證明社群持續信任；MCP 協議原生支援意味著與 Claude/Anthropic 生態工具（如本週 #1 的 claude-context）可無縫協作；生產級護欄設計適合需要穩定性的 Biotech 數據管線。
