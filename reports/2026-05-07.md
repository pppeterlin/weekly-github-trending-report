## 三、Project Detailed Introduction

### 1. [czlonkowski/n8n-mcp](https://github.com/czlonkowski/n8n-mcp)（原排名：24）⭐ 推薦分數：9.6

- **核心功能**：這是一個 MCP 伺服器，讓 Claude Desktop、Claude Code、Cursor、Windsurf 等 AI 工具可以直接在對話中呼叫 n8n 工作流引擎的 API。使用者可以直接對 Claude 說「幫我建立一個抓取 GitHub Trending 後寄 Email 的工作流」，Claude 透過 MCP 完成整個流程。
- **推薦原因**：完美橋接 AI Agent 與工作流自動化兩大技術棧。對於已使用 n8n 進行數據管道管理的工程師，這個工具讓 Agent 能直接操作現有的工作流基礎設施，大幅提升自動化能力。
- **技術亮點**：MCP 協議標準實作、n8n REST API 完整封裝、支援 OAuth 認證流程；20k stars 的爆發式成長顯示 MCP 生態已進入快速採納期。

### 3. [VectifyAI/PageIndex](https://github.com/VectifyAI/PageIndex)（原排名：27）⭐ 推薦分數：9.3

- **核心功能**：提出「無向量、推理驅動的 RAG」新正規化。PageIndex 用大型語言模型對文件建立結構化索引（而非 embedding 向量），在 SimpleQA 基準上達到 ~95% 精度。
- **推薦原因**：直接對應個人記憶引擎（Memory & RAG）評分維度（權重 20%）。對於需要在生物信息文獻、數據工程文件中進行精準問答的場景，這種無向量方法可顯著降低基礎設施複雜度。
- **技術亮點**：LLM 驅動的文件索引替代 embedding、推理時動態定位相關頁面、無需向量數據庫即可實現高精度 RAG。
